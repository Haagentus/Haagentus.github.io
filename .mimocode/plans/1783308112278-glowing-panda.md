## 二、jsDelivr CDN技术评估

### 2.1 jsDelivr工作原理
- **GitHub文件CDN**：`https://cdn.jsdelivr.net/gh/user/repo@version/file`
- **npm包CDN**：`https://cdn.jsdelivr.net/npm/package@version/file`
- **版本控制**：支持指定版本或使用latest
- **缓存机制**：全球多CDN节点，自动缓存

### 2.2 适用场景
- ✅ 公开的GitHub仓库
- ✅ 静态资源文件（图片、字体、JS、CSS）
- ✅ 需要全球加速访问的项目
- ✅ 开源项目

### 2.3 当前项目适用性
- **仓库类型**：GitHub Pages静态站点（公开仓库）
- **资源类型**：静态图片文件
- **访问需求**：需要全球加速（特别是中国地区）
- **结论**：**完全适用**

## 三、实施方案设计

### 3.1 CDN地址格式
```
https://cdn.jsdelivr.net/gh/Haagentus/Haagentus.github.io@main/imgs/001.webp
```

### 3.2 需要修改的文件
1. **JavaScript文件**：
   - `js/layout.js`：修改背景图数组路径

2. **CSS文件**：
   - `tools/genshinEventSourceDL/static/index.css`：修改背景图路径

3. **HTML文件**：
   - `tools/genshinEventSourceDL/index.html`：修改favicon路径

4. **Markdown文件**：
   - 10个包含图片引用的md文件

### 3.3 修改策略
#### 方案A：直接替换（简单）
- 直接将相对路径替换为CDN完整URL
- **优点**：改动小，风险低
- **缺点**：路径硬编码，维护困难

#### 方案B：配置化（推荐）
- 创建配置文件定义CDN基础URL
- 代码中动态拼接完整路径
- **优点**：易于维护，支持本地/CDN切换
- **缺点**：改动较大

### 3.4 推荐实施方案B
```javascript
// 配置文件：js/config.js
const CDN_CONFIG = {
  enabled: true,
  baseUrl: 'https://cdn.jsdelivr.net/gh/Haagentus/Haagentus.github.io@main',
  fallbackToLocal: true
};

// 使用示例：js/layout.js
function getImageUrl(path) {
  if (CDN_CONFIG.enabled) {
    return `${CDN_CONFIG.baseUrl}/${path}`;
  }
  return path;
}
```

## 四、风险评估

### 4.1 技术风险
- **CDN服务不可用**：jsDelivr有SLA保证，但仍有小概率故障
- **缓存更新延迟**：更新图片后CDN缓存可能需要时间刷新
- **版本管理**：需要确保CDN地址中的版本号与仓库分支/标签一致

### 4.2 缓解措施
- 实现本地回退机制（`fallbackToLocal`）
- 使用版本号而非分支名（如`@v1.0.0`）
- 定期检查CDN可用性

### 4.3 性能影响
- **正面影响**：全球加速，减少GitHub服务器压力
- **负面影响**：额外DNS解析时间（可忽略）
